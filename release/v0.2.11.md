# PDU Data Automation v0.2.11

This release includes fixes for bundled resource loading and operator folder selection flows, plus agent release documentation improvements.

## Changes

- **fix(config): load bundled layout resources at runtime (Issue 4)**:
  - Tauri app setup now registers `app.path().resource_dir()` at startup.
  - Added shared `resource_paths` module for consistent resolution of packaged configs.
  - Prioritizes release bundle layout (`_up_/config/report-layouts/`) with tolerant fallbacks for dev/packaging variations.
  - Wired `load_layout_profile()` and `load_accuracy_thresholds()` (and accuracy) to use resource candidates before dev paths and builtins.
  - Added tests that simulate bundled resources and verify they are preferred over `include_str!` defaults.
  - Updated docs to reflect the fix and note need for real installed-build validation.

- **fix(operator): setup reports on folder selection**:
  - "Browse unit folder" (chooseUnitFolder) now immediately triggers full report setup via `setupUnitFolder` (template copy from profile root, initial summary).
  - Previously some paths relied on lighter `scanUnitFolder`, which didn't prepare reports.
  - Adjusted flow: setup happens on selection (with empty SN if needed); Transformer SN save is now a distinct step.
  - Refactored CSV discovery in `build_summary_with_profile` into `CsvScanIndex` for single-pass scanning (performance + correctness for setup paths).
  - Updated tests and UI expectations to cover the new flow, batch usage, etc.

- **chore(release): rotate updater signing key**:
  - New signing key generated for future releases.

- **docs: capture release runbook for agents**:
  - Expanded `AGENTS.md` with release/updater notes, signing key location, asset naming (dot vs space), S-drive layout, validation commands, and known script quirks.
  - Detailed `docs/RELEASE_AND_DEPLOYMENT.md` with current v0.2.10+ practices, GitHub assets, S-drive structure (`release-support/vX.Y.Z/`, archive/), local signing helper script, and troubleshooting (e.g. NSIS signing errors).
  - Minor updates to CODE_REVIEW.md and CONFIGURATION_MODEL.md.

- Selecting a unit folder via Browse immediately performs report setup (templates copied if needed).

- Resource and operator fixes do not regress existing setup, batch processing, or profile-driven reports.
- Docs reflect current release process.
