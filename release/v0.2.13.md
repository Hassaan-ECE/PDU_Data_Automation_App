# PDU Data Automation v0.2.13

Operator settings UX polish, end-of-shift summary on the Settings home, and multi-station early post.

## Changes

- **feat(settings): cleaner two-level settings chrome**
  - Settings home: **Shifts**, **Summary options**, shield **Advanced** icon (top-right, password-gated).
  - Arrow-only back control; centered page titles per submenu.
  - Unsaved-leave dialog: **Save Changes** / **Discard Changes** / **Cancel** (click outside cancels).

- **feat(settings): Shifts**
  - Single/double shift mode with improved desktop time pickers.
  - Shift windows kept consistent for double-shift schedules.

- **feat(settings): Summary options**
  - Station include checklist with **Main** poster picker on each row.
  - Enable toggle on the save row (summary posts on by default).

- **feat(settings): End of shift on Settings home**
  - Full preview panel + **Post Summary** under it (any station may post early).
  - Confirm before post; shared log records who posted so other PCs skip a second post.
  - Summary card no longer includes a **Posted by** line.

- **feat(notifications): backend summary for any station**
  - Manual post no longer limited to the designated poster station.
  - Shared `shift_log` tracks last summary time/station/shift for floor awareness.

## Not in this release

- Automatic scheduled end-of-shift posts at shift end times.
- Stuck / idle automation.

## Validation

- Frontend Vitest (settings, operator panel, shift range picker).
- Backend notification unit tests including floor summary.

## Deploy notes for station PCs

1. Install or update via app updater / `PDU Data Automation_0.2.13_x64-setup.exe` from the S-drive or GitHub Release.
2. Confirm Settings: shield → Advanced still password-gated; End of shift preview on home.
3. After first early **Post Summary**, other stations should see “Already posted” until new events log.
