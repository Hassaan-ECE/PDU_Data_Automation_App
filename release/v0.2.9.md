# PDU Data Automation v0.2.9

This release adds the manual Print Report flow for the pilot app.

## Changes

- Add manual Print Report flow.
- Replace the single full-width Open Report action with side-by-side Open Report and Print Report buttons.
- Block Print Report while the runner is active, while report setup is missing, and while Transformer SN is missing or unsaved.
- Add operator confirmation modal before printing.
- Persist editable operator names in frontend localStorage, seeded with Sean, Long, and Jose.
- Support typed operator names, case-insensitive dedupe, autocomplete suggestions, and removing saved names from the dropdown.
- Write final operator name as text to the print report workbook at `Test Report #2!E39`.
- Open Excel's non-silent print UI for operator confirmation.
- Select all visible worksheets before opening print UI so the whole print workbook is targeted.
- Bring Excel to the foreground when opening the print UI.
- Return clearer backend errors for missing, locked, or malformed print report workbooks and print dialog failures.

- Print Report writes the operator name to the correct cell.
- Excel print UI opens for operator confirmation.
- Excel comes to the foreground.
- The flow does not silently print.
