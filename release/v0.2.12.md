# PDU Data Automation v0.2.12

Operator Teams notifications integrated into the main app for floor pilot testing.

## Changes

- **feat(notifications): Teams operator alerts in the main app**
  - Background worker posts Problem and Complete Adaptive Cards without blocking CSV/Excel automation.
  - Durable receipts in `unit_state.json` reduce duplicate cards; Problem receipts clear when a task later passes.
  - Soft-fail only: missing config, queue full, Teams failure, or shared-log errors never fail the automation path.

- **feat(settings): password-gated in-app notification settings**
  - Cog → factory password `0601` → Settings (re-locks on Back).
  - Configure station (1–4), destination label, webhook URL, enabled flag, and change password.
  - Settings stored under the app config dir (`notification_settings.json`), not hand-edited JSON beside the exe.
  - Test ping from Settings with result correlation.
  - Settings cog disabled while automation / setup / print prompts are active.

- **feat(notifications): shared OneDrive folder for multi-station log**
  - Browse to a shared folder (e.g. hidden OneDrive folder) in Settings.
  - On Save, creates `shift_log.json` and `stations/test-station-1…4` under that folder.
  - Problem/Complete appends to the shared log after Teams accepts the card (best-effort).

- **ui:** remove Teams status line from the operator panel footer (Settings/Test ping remain the control surface).

- **docs:** `docs/NOTIFICATIONS.md` and design/plan notes for the notification settings slice.

## Not in this release

- End-of-day / multi-shift scheduled summary posting (planned next).
- Stuck / idle automation (deferred for burn-in false positives).

## Validation

- Local desktop smoke (`bun run desktop`): settings unlock, station/webhook, Test ping, Problem card, shared folder layout, footer clean.
- Automated: notification Rust tests, frontend Vitest including settings/password/shared-folder UI.

## Deploy notes for station PCs

1. Install `PDU Data Automation_0.2.12_x64-setup.exe` (manual if updater not yet trusted from older builds).
2. Cog → `0601` → set **Test Station N**, paste Teams Workflow URL, enable, Save.
3. Send **Test ping** and confirm the PDU Testing chat.
4. Optionally Browse the shared OneDrive notifications folder (same path on every station) and Save.
5. Change the settings password after first setup if desired.
