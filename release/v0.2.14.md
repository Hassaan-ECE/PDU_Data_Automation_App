# PDU Data Automation v0.2.14

Shared floor notification settings: multi-PC sync via the shared OneDrive folder, editable station names, and safer Connect/adopt behavior.

## Changes

- **feat(notifications): shared `floor_settings.json`**
  - When Advanced → Browse points at the shared notifications folder (e.g. `.PDU_Notifications`), floor-wide settings live in that folder and sync across stations.
  - Shared: password, webhook, destination, event toggles, shifts, Main poster, included stations, station **display names**.
  - Local only: this PC’s station id and the path to the shared folder.

- **feat(settings): scoped saves**
  - Operator saves: shifts / Main / included stations only.
  - Advanced saves: webhook, names, this-PC station, toggles.
  - Connect: adopt existing floor or seed a new one (does not overwrite an existing floor with a stale form).
  - Local: this-PC station and clearing the shared path.

- **feat(settings): Connect floor password**
  - After Browse to a new/changed shared folder, enter **Existing floor password** when the floor password differs from this PC’s Advanced unlock password.
  - First seed can include Advanced form fields (webhook/names) on the same Connect save.

- **feat(settings): station display names**
  - Advanced → Station & Teams: edit labels for the four fixed slots without a code release.
  - Labels sync via the floor file; Teams cards and summary use the shared catalog.

- **fix: no auto-reseed / catalog cache / RMW lock**
  - Ordinary loads never recreate a missing floor file from incomplete cache.
  - Full station catalog cached locally for offline labels.
  - Floor write uses a cross-process lock around read–patch–write (OneDrive multi-PC remains best-effort).

- **ui: Settings open poll**
  - While Settings is open and the form is clean, reload floor/local settings about every 45 seconds.
  - Dirty forms are not clobbered by peer updates.

## Deploy notes for station PCs

1. Install `PDU Data Automation_0.2.14_x64-setup.exe` (S-drive or GitHub).
2. On each station: Advanced → Browse the **same** synced `.PDU_Notifications` folder.
3. If the floor already has a non-default password, enter it in **Existing floor password**, then Save (Connect).
4. Set **This PC station** correctly; save Advanced renames/webhook from one admin PC when needed.
5. Operators can still change Main / shifts without the Advanced password; those sync when the shared folder is set.
