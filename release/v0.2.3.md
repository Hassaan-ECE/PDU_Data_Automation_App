# PDU Data Automation v0.2.3

This release is intended to test the in-app updater from `v0.2.2` on the operator PC during an active ATS run.

## Changes

- Keep the runner in `waiting` when ATS is still writing the current CSV instead of showing an I/O-bound processing failure.
- Wait for the legacy per-step duration after a current-step CSV first appears before processing it.
- Keep backlog processing fast for already-completed detected CSVs.
- Add scan metadata for the latest detected CSV start time so the frontend can make the wait decision.
