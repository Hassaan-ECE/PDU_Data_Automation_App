# PDU Data Automation v0.2.4

This release is intended to test starting the app while ATS is already partway through a step, including system burn-in.

## Changes

- Treat detected CSVs with nominal time remaining as the current in-progress task, not skipped backlog.
- Resume the countdown from the CSV timer start time when the app starts mid-step.
- Report latest CSV readability in folder scans so locked active CSVs keep the runner waiting.
- Use the STEP71 CSV as the System Burn-In countdown start even when STEP72 report data is also present.

## Validation

- `node scripts\run-bun.mjs run lint`
- `node scripts\run-bun.mjs run test`
- `node scripts\run-bun.mjs run build`
