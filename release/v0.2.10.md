# PDU Data Automation v0.2.10

This release delivers a significant performance improvement for processing previously-detected tests (the "Run Previous Tests" / backlog flow) along with further data-driven configuration work.

## Changes

- **Batched previous-test report writes (major speed win)**: 
  - New `process_automation_tasks` backend command (and Tauri command) for batch processing of multiple tasks.
  - Computes task results and collects Excel patches without side effects.
  - Aggregates patches by target workbook path.
  - Performs a single transactional commit per workbook (via existing `patch_workbooks_transactional`) at batch boundaries.
  - Records results to `unit_state.json` only after the associated workbook commit succeeds.
  - Emits `automation-task-batch-progress` events with per-task index/total, state, and message for live UI updates.
  - Idempotent ("already processed from the same CSV") tasks are short-circuited without re-work.
  - On commit failure, successfully committed prior items in the group are recorded; the failing item is turned into a clear "Report commit failed" result; processing stops.
  - "Run Previous Tests" in the UI now uses the batch path instead of looping single-task calls.

- Profile-driven report filenames and discovery:
  - `templates.main_report_template` and `templates.print_report_template` now control the names of templates copied from the root.
  - `workbooks.main.file_pattern` and `workbooks.print.file_pattern` now drive `discover_reports`, `require_*_report`, and related lookup (with legacy fallbacks when omitted).
  - `ReportFileConfig` + helpers (`report_file_config`) centralize this; legacy constants kept only for defaults and tests.

- Invalid/unreadable layout profiles now fail visibly (no silent fallback) in setup, scan, process, print readiness, etc. (builds on earlier profile error surfacing work).

- Refactors supporting the above: separation of `compute_task*` (pure, returns `ProcessorTaskOutput` with result + patches) from commit/record, threading of `ReportFileConfig` through processors/mapped/batch paths, new plural `record_processor_results` under lock in unit_state.

- Test coverage: new batch processing tests (success, commit failure without recording tentative passes, partial prior recording on mixed success/failure, verification failures after some successes), report file config tests with custom names/patterns, profile validation requiring workbook entries.

- Frontend: `processAutomationTasks`, `listenAutomationTaskBatchProgress`, `TaskBatchProcessResult` / `TaskBatchProgress` types and mock; OperatorPanel `runTaskBatch` path for backlog, progress listener, result application; updated tests verify batch is used for previous tests.

- Miscellaneous: docs updates (CODE_REVIEW, ARCHITECTURE, README status), frontend test tweaks, continued cleanup.

## Validation

- `bun run lint`
- `bun run test`
- `bun run build`
- `bun run check:versions`
- `bun run validate:report-layouts`
- `cargo fmt --manifest-path backend\Cargo.toml --check`
- `cargo check --manifest-path backend\Cargo.toml`
- `cargo test --manifest-path backend\Cargo.toml`

Manual validation was completed before the release request:

- "Run Previous Tests" with several detected steps is dramatically faster (one Excel commit per workbook instead of per task).
- All reports receive the correct cell updates.
- State in unit_state.json reflects only successfully committed work.
- Failures during batch (e.g. locked report) produce clear errors, record prior successes, and do not leave partial/inconsistent state.
- Single-task flow, profile-driven names, and error paths for bad profiles continue to work.
- Existing functionality (setup, individual steps, print report, etc.) is unaffected.

## Caveats

- Operator-machine install and real updater upgrade smoke tests are still pending (carry-over from prior releases).
- The `SN##` serial substitution rule remains an implicit convention inside `ReportFileConfig::main_report_name` (profile templates contain `SN##` and the code does a simple replace). Future hardening item.
