# PDU Data Automation v0.2.17

Operator recovery release: failed readings stay visible and red without stopping the test, batch runs finish the remaining available work before returning to the first failure, and Excel recovery is more direct and reliable.

## Changes

- **Keep testing after an accuracy failure**
  - Out-of-range readings are still written into Excel and the task remains failed/red.
  - Automatic processing continues to later ready steps instead of pausing the app at the bad reading.
  - Batch processing completes all available steps, then returns focus to the first failure.
  - Once a failure occurs, automatic scrolling stops following later steps so the operator can see the actionable failure when returning to the station.

- **Improve failure review and operator decisions**
  - Open from a failed step now targets the exact report workbook, worksheet, and failed cell.
  - Opening a report no longer requires a Transformer SN; the SN is still required before printing.
  - The former Skip action is now Pass, allowing an operator to accept a negligible result for the current run while preserving normal validation on future reruns.

- **Recover from an open Excel report**
  - When Excel is locking the report, the app offers to close only that report workbook and retries the pending operation.
  - Other Excel workbooks remain open.
  - Report lock failures from task processing now use the same targeted recovery flow instead of appearing as a generic Processing Error.

- **Add a safe floor simulation**
  - `bun run desktop` continues to launch the real app with live CSV and Excel processing.
  - `bun run desktop:demo` launches an accelerated simulation with a deliberate failed reading and no real CSV or workbook changes.
  - Countdown and readiness status text remains stable during simulated progression.

## Operator test

1. Install `PDU Data Automation_0.2.17_x64-setup.exe` on the operator test PC.
2. Run a safe unit with one known out-of-range reading and later STEP files already available.
3. Confirm the bad reading is written to Excel, the step stays red, and the remaining available steps continue.
4. Confirm the panel returns to and remains focused on the first failed step after the batch finishes.
5. Use Open on the failure and confirm Excel selects the failed worksheet and cell.
6. Leave only that report open in Excel, rerun a write, and confirm the app can close the report and retry without closing unrelated workbooks.
7. Confirm Pass clears the current blocking decision, while rerunning the same bad data still validates and fails normally.
