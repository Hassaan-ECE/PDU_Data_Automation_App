# PDU Data Automation v0.2.7

This release adds the Start-time Unit Setup flow for Transformer SN capture.

## Changes

- Open a Unit Setup dialog when Start is pressed for a new or reset unit session.
- Suggest the newest likely ATS-created unit folder/SN and allow `...` browsing to another unit folder.
- Require Transformer SN before setup can continue.
- Write Transformer SN as text to the main report workbook at `Test Summary!D1`.
- Keep existing previous-tests/start behavior after setup succeeds, including the previous-tests prompt when completed CSVs are already detected.
- Return clear setup errors for blank Transformer SN, locked workbooks, missing main reports, and missing report sheets.

## Validation

- `node scripts\run-bun.mjs run lint`
- `node scripts\run-bun.mjs run test`
- `node scripts\run-bun.mjs run build`
- `cargo fmt --manifest-path backend\Cargo.toml --check`
- `cargo check --manifest-path backend\Cargo.toml`
- `cargo test --manifest-path backend\Cargo.toml`
