# PDU Data Automation v0.2.16

Accuracy-failure report preservation: out-of-range system and breaker readings are now written into Excel while the task remains failed and the existing operator workflow stays gated.

## Changes

- **fix(automation): preserve failed accuracy data in the report**
  - Built-in 208V and 415V system/breaker processors retain their prepared workbook updates when threshold verification fails.
  - Meter values, detected values, and calculated accuracy values are written so the report records the actual bad result instead of leaving the step blank.
  - Direct/manual reruns and automatic batch processing both commit the failed step's data before returning the failure.

- **safety: keep failure handling unchanged**
  - The task still returns and persists as `fail`, including the existing failed-cell location and threshold details.
  - Automatic processing still stops at the failed step, and Print Report remains blocked until the failure is resolved or explicitly accepted.
  - Missing, blank, malformed, or unreadable CSV values still fail without being converted into valid-looking zeroes.
  - Workbook write errors still surface as report commit failures rather than hiding the original file problem.

## Operator test

1. Install `PDU Data Automation_0.2.16_x64-setup.exe` on the operator test PC.
2. Process a safe test unit containing a known out-of-range system or breaker reading.
3. Confirm the step is still visibly failed and automatic processing stops at that step.
4. Open the main report and confirm the meter, detected, and accuracy cells contain the failed readings.
5. Correct or accept the failed step using the existing operator workflow, then continue the unit.
