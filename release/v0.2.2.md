# PDU Data Automation v0.2.2

This release is intended to test the in-app updater from `v0.2.1`.

## Changes

- Change Reset into a three-step flow when a unit folder is selected:
  - first reset collapses expanded test groups
  - second reset keeps the selected SN folder and resets task states from detected data
  - third reset clears the selected SN folder back to the startup-like state
- Update the Reset button label to show the next action: `Collapse Tests`, `Reset Current SN`, or `Clear SN`.

## Validation

- `node scripts\run-bun.mjs run lint`
- `node scripts\run-bun.mjs run test`
- `node scripts\run-bun.mjs run build`
- `cargo fmt --manifest-path backend\Cargo.toml --check`
- `cargo check --manifest-path backend\Cargo.toml`
- `cargo test --manifest-path backend\Cargo.toml`
