# PDU Data Automation v0.2.5

This release is intended to test the operator timer while starting mid-test and then processing previous detected steps first.

## Changes

- Use one total countdown calculation instead of mixing total-step time and active-step time.
- Count active detected/waiting CSV time plus future unpassed step time.
- Keep already completed detected CSVs from adding test duration while still requiring the task to pass before the runner advances past it.
- Remove direct active-step timer overwrites that caused the display to bounce between partial totals.
