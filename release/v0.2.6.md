# PDU Data Automation v0.2.6

This release starts the production data-driven report mapping migration with the transformer checks.

## Changes

- Add a generic mapped report processor path driven by `config/report-layouts/pdu500.rev02.layout.json`.
- Run 208V and 415V transformer report writes from production layout mappings instead of hardcoded transformer cell constants.
- Keep the existing Rust processors as fallback for every task family that has not been migrated yet.
- Add config/runtime support for explicit row numbers, first data row after header, last data row after header, last data row, and last numeric row selectors.
